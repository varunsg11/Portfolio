Section 01

Overall Architecture

The project is split into two independent parts — a static frontend and a Python backend. They communicate over HTTP. This is called a decoupled architecture.

┌─────────────────────────────────────────────────────────────┐ │ USER'S BROWSER │ └───────────────────────────┬─────────────────────────────────┘ │ visits URL ▼ ┌───────────────────────────────────────────────────────────┐ │ NETLIFY (Frontend Host) │ │ serves index.html + style.css + script.js │ └───────────────────────────┬───────────────────────────────┘ │ ┌─────────────────┴────────────────┐ │ contact form POST │ resume download GET ▼ ▼ ┌─────────────────────────────────────────────────────────┐ │ RENDER (Backend Host) │ │ FastAPI · Python · uvicorn │ │ │ │ /api/contact ──► validates ──► Resend API │ │ /resume ──► serves PDF file │ │ /health ──► returns {"status":"ok"} │ └────────────────────────────┬────────────────────────────┘ │ ▼ ┌──────────────────────────────┐ │ RESEND (Email Service) │ │ delivers email to Gmail │ └──────────────────────────────┘

Codebase — File by File

What each frontend file does

index.html — The skeleton. Every section (hero, about, experience, etc.) is written as HTML here. It links to style.css and script.js. Think of it as the bones.

style.css — The appearance. Colors, fonts, spacing, animations, responsive breakpoints. Uses CSS custom properties (--accent, --bg etc.) defined in :root so one change updates the whole site.

script.js — The behaviour. Three jobs: (1) fade-up scroll animations using IntersectionObserver, (2) mobile hamburger nav toggle, (3) contact form — intercepts submit, POSTs JSON to the backend, shows success/error.

What main.py does

main.py — The entire backend in one file. Three endpoints:

  • POST /api/contact — receives form data, validates it with Pydantic, sends email via Resend
  • GET /resume — serves the PDF file as a download
  • GET /health — returns OK (used to check if server is running)

Section 03

Contact Form — End to End Flow

When someone fills in the contact form and clicks Send, here is exactly what happens step by step:

1

User clicks Send

script.js intercepts the form submit event with e.preventDefault() — this stops the page from refreshing, which is the browser's default behaviour.

2

JavaScript builds a JSON payload

Reads the form field values (name, email, subject, message) and packages them as a JSON object.

3

fetch() sends a POST request to Render

The browser makes an HTTP POST request to https://portfolio-9s06.onrender.com/api/contact with the JSON in the request body.

4

FastAPI receives the request

The /api/contact endpoint is triggered. Pydantic automatically validates the data — checks email format, checks no fields are empty. If invalid, returns a 422 error immediately.

5

Backend calls Resend API

FastAPI calls Resend's API with your API key, the sender address, your Gmail as recipient, and an HTML email body containing the message.

6

Resend delivers the email

Resend's servers send the email to varunsg118@gmail.com. FastAPI returns {"message": "Sent"} to the browser.

7

Frontend shows success message

script.js reads the response, sees status 200, and shows the green "Message sent!" banner. If anything failed, shows the red error banner instead.

IntersectionObserver — how scroll animations work

// Watches elements. Fires when they enter the viewport. const observer = new IntersectionObserver((entries) => { entries.forEach(entry => { if (entry.isIntersecting) { entry.target.classList.add('visible'); // triggers CSS transition observer.unobserve(entry.target); // stop watching once done } }); }, { threshold: 0.12 }); // fires when 12% of element is visible // CSS does the actual animation: // .fade-up { opacity:0; transform:translateY(28px); transition:... } // .fade-up.visible { opacity:1; transform:translateY(0); }
